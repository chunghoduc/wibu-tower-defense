# Achievement SDXL Icons — Design Spec (2026-06-14)

## Problem
The achievements board renders text-only cards with an ugly 🏆 emoji and no per-achievement
art. The user wants real icons generated through the **SDXL flow** (`scripts/sdart/`), the
SOLE art pipeline — "always use sdxl flow for icons".

## Decision
Give **each of the 20 achievements its own distinct SDXL emblem**, rendered on the left of its
card. Visual language = a polished **circular trophy medallion** (beveled metal rim + raised
embossed motif), tier-metalled (bronze → silver → gold) so prestige reads at a glance. This is
deliberately richer than the flat `roleicon` glyphs because achievements are *rewards/badges*.
Full-colour (NOT tinted at render — unlike roleicon), so each medal carries its own accent.

## Approach (mirrors the roleicon precedent end-to-end)
1. **prompts.mjs** — add `ACHIEVEMENT_VISUAL` (id → medallion motif), `ACHIEVEMENT_NEGATIVE`,
   and `achievementIconStyle(visual)`. Declared in `prompts.d.mts`.
2. **assetKeys.ts** — add `achievementTex(id) => \`achievement__${id}\`` (sole key builder).
3. **sdgen.mjs** — `buildJobs()` loops `ACHIEVEMENT_VISUAL`, `kind:"achievement"`, 768×768 → cut 128.
   Generated by `npm run gen:sprites --only=achievement` into `public/assets/sprites/achievement/`.
4. **PreloadScene** — `load.image(achievementTex(id), versioned(...))` for every catalog id.
5. **AchievementScene.drawCard** — draw a 64px medallion at the card's left, shift text right.
   Graceful fallback: only add the image when `this.textures.exists(key)` (missing → no icon, no
   green __MISSING box).

## Drift guard (TDD)
`tests/achievementIconPrompts.test.ts` — `ACHIEVEMENT_VISUAL` keys must equal the `ACHIEVEMENTS`
catalog ids exactly (none missing, none dead), each description non-empty and distinct.
`assetKeys` test extended for `achievementTex`.

## Non-goals
No `ASSET_VERSION` bump rule change (this DOES regenerate art → bump on deploy). No new mechanic,
no reward/progress change, no manifest entry (loaded directly like roleicon, not a spritesheet).
