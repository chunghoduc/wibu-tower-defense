# Enemy Walk Animations + Animated VFX — Design Spec

**Date:** 2026-06-07
**Status:** Approved

---

## Goal

Replace all static single-frame enemy and VFX sprites with animated multi-frame sprite sheets so:
- Enemies visibly walk/move across the battlefield instead of floating
- Skill visual effects play a full lifecycle animation instead of flashing a static image

---

## Scope

### In scope
- All 16 enemy sprites → 3-frame walk+hurt strip
- All 12 VFX sprites → 6-frame lifecycle animation strip
- `wibu-td-designer` generator updated to produce multi-frame strips
- Game sprite manifest updated with new frame counts and names
- Three small game-code changes (PreloadScene, BattleScene, fx.ts)

### Out of scope
- Enemy attack animation (requires battle-event wiring — separate feature)
- Boss animations (already have 7 frames; no change needed)
- Tower/hero animations (no change)
- New enemy archetypes or VFX types

---

## Asset Formats

### Enemy strips (3 frames)

```
PNG layout:  [walk1][walk2][hurt]
Width:       frameWidth × 3   (frameWidth is per-enemy, e.g. brute=160, slimelet=88)
Height:      frameWidth       (square frame, unchanged from today)
```

| Frame | Action description |
|-------|--------------------|
| `walk1` | First stride — weight shifting forward, mid-motion, one foot raised |
| `walk2` | Opposite stride — weight on back foot, arms/limbs swinging opposite |
| `hurt`  | Recoil — knocked off-balance, pain expression, knocked backward |

### VFX strips (6 frames)

```
PNG layout:  [spawn][grow][peak][hold][fade][gone]
Width:       768px  (6 × 128)
Height:      128px
```

| Frame | Action description |
|-------|--------------------|
| `spawn` | Small seed — just appearing, minimal energy, compact |
| `grow`  | Building — energy gathering, expanding outward |
| `peak`  | Maximum — full burst, brightest frame, most particles |
| `hold`  | Sustained — slightly diffused, held at near-peak |
| `fade`  | Dissipating — energy scattering outward, dimming |
| `gone`  | Final wisps — nearly transparent, trailing particles only |

## Generator Changes (`wibu-td-designer/src/gen-all.mjs`)

### Enemy generation

`runEnemies()` generates **3 images per enemy** using the same base prompt as today, with per-frame action suffixes:

```
walk1 → ", first walk stride, weight shifting forward, mid-motion, one foot raised"
walk2 → ", opposite walk stride, weight on back foot, opposite limbs forward"
hurt  → ", recoil from hit, knocked backward, pain expression, off-balance"
```

Post-processing: pack 3 `frameSize×frameSize` frames into a `(frameSize×3)×frameSize` horizontal strip.

Seeds: `baseSeed + frameIndex` (same seed discipline as towers).

### VFX generation

`runVfx()` generates **6 images per VFX** using a lifecycle-aware base prompt, with per-frame suffixes:

```
spawn → ", small seed form, minimal energy, compact nascent state"
grow  → ", energy building, expanding outward, gathering force"
peak  → ", maximum intensity, full burst explosion, brightest most saturated"
hold  → ", sustained intensity, slightly diffused, holding peak energy"
fade  → ", dissipating outward, energy scattering, dimming trails"
gone  → ", final wisps only, nearly transparent, last fading traces"
```

Post-processing: pack 6 `128×128` frames into a `768×128` horizontal strip.

## Game Code Changes (3 files)

### 1. `src/scenes/PreloadScene.ts` — Add VFX animation pattern

Location: inside the animation-creation loop, after the existing `mk()` calls.

Add one line:
```typescript
mk("vfx", idx(/spawn|grow|peak|hold|fade/), 12, 0);
```

This creates animation key `vfx__<id>_vfx` for every VFX entry, playing once at 12 FPS (~500ms for 6 frames).

The existing `_walk` and `_hurt` patterns already match `walk1/walk2` and `hurt` frame names — no further change needed for enemies.

### 2. `src/scenes/BattleScene.ts` — Play walk animation on enemies

Location: `ensureSprite()` method, the animation play block at the end.

```typescript
// Before:
if (this.anims.exists(`${key}_idle`)) s.play(`${key}_idle`);

// After:
const animKey = this.anims.exists(`${key}_walk`) ? `${key}_walk` : `${key}_idle`;
if (this.anims.exists(animKey)) s.play(animKey, true);
```

Enemies with walk frames play `_walk` (loops forever via `repeat: -1`). Anything without falls back to `_idle`. No other enemy-rendering code changes.

### 3. `src/scenes/fx.ts` — Animate VFX sprites on skill burst

Location: `skillBurst()` lines 281-285. Currently uses `this.fac.image()` (static) + a tween.

Replace with an animated sprite that self-destructs on animation complete:

```typescript
// Before (lines 281-285):
const key = skillId ? `vfx__${skillId}` : "";
if (key && this.scene.textures.exists(key)) {
  const spr = this.fac.image(at.x, at.y, key).setDepth(this.depth + 3).setScale(0.3).setAlpha(0.95);
  this.scene.tweens.add({ targets: spr, scale: 1.7, angle: 60, alpha: 0, duration: 460, ease: "Cubic.easeOut", onComplete: () => spr.destroy() });
}

// After:
const key = skillId ? `vfx__${skillId}` : "";
if (key && this.scene.textures.exists(key)) {
  const animKey = `${key}_vfx`;
  if (this.scene.anims.exists(animKey)) {
    const spr = this.scene.add.sprite(at.x, at.y, key)
      .setDepth(this.depth + 3).setScale(1.0).setAlpha(0.95);
    spr.play(animKey);
    spr.once("animationcomplete", () => spr.destroy());
  } else {
    // Fallback: original tween for any VFX still missing animation data
    const spr = this.fac.image(at.x, at.y, key).setDepth(this.depth + 3).setScale(0.3).setAlpha(0.95);
    this.scene.tweens.add({ targets: spr, scale: 1.7, angle: 60, alpha: 0, duration: 460, ease: "Cubic.easeOut", onComplete: () => spr.destroy() });
  }
}
```

Note: `this.fac` is `this.scene.add` in the FxLayer context — the sprite must be created via `this.scene.add.sprite()` (not `this.fac.image()`) to support animation playback.
